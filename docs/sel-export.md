# Sel Export — «Тезкор маълумот» по водохранилищам в период паводков

Excel/PDF отчёт почасового состояния включённых в `reservoir_flood_config` водохранилищ. Подаёт сравнение «час−1 / час» по каждому параметру (уровень, объём, келиш, чиқиш, ГЭС орқали, мощность, салт ташлама), плюс погоду, температуру и ФИО дежурного.

## Endpoint

```
GET /reservoir-flood/export?date=YYYY-MM-DD&hour=HH&format=excel|pdf
```

| Параметр | Обязательный | Default | Описание |
|---|---|---|---|
| `date` | да | — | Дата отчёта (YYYY-MM-DD) в Asia/Tashkent. |
| `hour` | нет | `0` | Час среза (0..23). См. раздел «Семантика prev / curr» ниже. |
| `format` | нет | `excel` | `excel` или `pdf`. |

**Доступ**: `sc` + `rais`. Роль `reservoir_duty` к этому endpoint'у не имеет доступа (она ограничена одной организацией, а отчёт — сводный по всем включённым).

**Имя файла**: `ТЕЗКОР-МАЪЛУМОТ-YYYY-MM-DD-HH.{xlsx|pdf}`.

**Рабочее окно**: 21..08 (вечер–утро, через полночь). Endpoint технически принимает любой час; запрос с `hour ∉ {21..23, 0..8}` обрабатывается, но в лог пишется WARN — это сигнал, что запрос вне обычного режима.

## Источник данных

Отчёт читает только две таблицы:

1. `reservoir_flood_config` — список включённых организаций (`is_active=TRUE`) и порядок (`sort_order`).
2. `reservoir_flood_hourly` — две точки (prev, curr) на каждую включённую организацию (см. ниже).

Если для текущего часа записи нет — соответствующие ячейки в Excel показывают `"-"`. Если нет prev-записи — то же самое. Дельты в строке 7 (и аналогах для остальных блоков) автоматически становятся `"-"` благодаря формуле `=IFERROR(D6-C6, "-")` в шаблоне.

## Семантика prev / curr

**curr** — запись со `recorded_at` ровно равным `date+hour` в Asia/Tashkent. Если для текущего часа записи нет, curr-стороны (D, F, …, P) и поля Q/R/S отображаются как `"-"`.

**prev** — самая свежая запись с `recorded_at < curr` для каждой организации индивидуально, **без верхнего ограничения** по возрасту. Это нужно, потому что наблюдения вводятся не на фиксированной сетке: ночью почасово, днём — раз в 3 часа, бывают пропуски, причём режимы разнятся между организациями. Запрос строится через `unnest($orgIDs) JOIN LATERAL (... LIMIT 1)` против индекса `(organization_id, recorded_at DESC)` — N коротких index seek'ов, по одному на организацию.

**duty_name fallback**: если curr.duty_name пуст, используется duty_name из prev. Семантическое последствие: при отсутствии записи curr в S может оказаться имя дежурного, записанное за дни/недели назад. Принято осознанно как «лучше показать предыдущего, чем пусто».

## Маппинг колонок шаблона (`template/sel.xlsx`)

Шаблон содержит один эталонный 2-рядный блок (ряды 6-7), который генератор клонирует под каждый резервуар. Подпись (E9:J9 хардкод + M9:Q9 для имени оператора) автоматически уезжает вниз.

| Колонка | Header (ряд 4) | Значения ряд 6 (час−1 / час) | Источник из БД |
|---|---|---|---|
| A | Т/р | порядковый номер | вычисляется генератором (1, 2, 3, …) |
| B | Сув омборлар номи | имя резервуара | `organizations.name` целиком |
| C/D | сатҳ, м +/− | level prev / curr | `reservoir_flood_hourly.water_level_m` |
| E/F | ҳажм, мlн.м³ +/− | volume prev / curr | `reservoir_flood_hourly.water_volume_mln_m3` |
| G/H | келиш, м³/с +/− | inflow prev / curr | `reservoir_flood_hourly.inflow_m3s` |
| I/J | чиқиш, м³/с +/− | outflow prev / curr | `reservoir_flood_hourly.outflow_m3s` |
| K/L | ГЭС орқали +/− | ges flow prev / curr | `reservoir_flood_hourly.ges_flow_m3s` |
| M/N | Қуввати +/− | capacity prev / curr | `reservoir_flood_hourly.capacity_mwt` |
| O/P | салт ташлама +/− | idle discharge prev / curr | `reservoir_flood_hourly.idle_discharge_m3s` |
| Q | Об-хаво холати | weather (current hour) | `reservoir_flood_hourly.weather_condition` |
| R | харорат | temperature °C (current hour); формат `+0;-0;0` — округление до целого со знаком | `reservoir_flood_hourly.temperature_c` |
| S | Диспетчер Ф.И.Ш | duty (current, fallback prev) | `reservoir_flood_hourly.duty_name` |

Ряд 7 (и клонированные `9, 11, …, 23`) — формулы дельты `=IFERROR(D6-C6, "-")` по парам колонок. Дельты имеют формат `+0,00;-0,00;0,00` — две цифры после запятой со знаком, ноль без знака. Excelize при `DuplicateRowTo` копирует формулу буквально (без сдвига ссылок), поэтому генератор после клонирования **перезаписывает** дельту корректным `=IFERROR(D{value_row}-C{value_row}, "-")` для каждого скопированного блока.

**Print_area** перепрошивается генератором динамически в `'<sheet>'!$A$1:$S$<signer_row>` — фактическая ширина таблицы под фактическое количество резервуаров.

## Параметризация по часу и подзаголовки row-5

В шаблоне:

- `S2` хранит время — `HH:00`. Генератор пишет `time.Time` с нужным часом.
- `S3` хранит дату.
- Подзаголовки D5/F5/H5/J5/L5/N5/P5 (curr) — время curr (`HH:00`), записывается генератором как Excel time-value, формат `hh:mm`.
- Подзаголовки C5/E5/G5/I5/K5/M5/O5 (prev) — **текстовый диапазон** фактических prev-времён по всем строкам отчёта (см. ниже).

### prev-подзаголовок: текстовая строка

Поскольку prev определяется per-org и не привязан к фиксированному часу, единого «часа prev» нет. Генератор рассчитывает диапазон `min..max` от всех непустых `PrevAt` и пишет одну строку во все семь prev-ячеек row-5. Стиль ячейки переключается на текстовый (`NumFmt = 49 = "@"`), чтобы LibreOffice под русской локалью не пытался интерпретировать дату как время.

Правила формирования (`formatPrevRow5` в [internal/lib/service/excel/sel/generator.go](../internal/lib/service/excel/sel/generator.go)):

| Случай | Результат | Пример |
| --- | --- | --- |
| ни у одной строки нет `PrevAt` | `"-"` | `-` |
| `min == max` и все на дату отчёта | одно значение `HH:00` | `14:00` |
| `min == max` и дата ≠ дате отчёта | одно значение `DD.MM HH:00` | `12.05 23:00` |
| `min ≠ max`, обе стороны на дату отчёта | `HH:00–HH:00` | `11:00–14:00` |
| `min ≠ max`, хотя бы одна сторона отличается от даты отчёта | `DD.MM HH:00–DD.MM HH:00` (дата на обеих сторонах) | `12.05 23:00–13.05 14:00` |

Разделитель — en dash `–` (U+2013), отличается от `"-"` (плейсхолдер «нет данных»). Дата на обеих сторонах вне-дневного диапазона нужна, чтобы ячейка читалась однозначно: иначе `12.05 23:00–14:00` визуально неясно, относится ли `14:00` ко вчера или к сегодня.

## Отсутствующие данные

Для всех ячеек блока (числовые M/N + Q/R/S и пары C/D…O/P): если значение `nil`/пусто — генератор пишет строку `"-"`. Дельты в ряду 7 благодаря IFERROR тоже становятся `"-"`. Сценарий «нет prev-записи» полностью покрыт: блок выводится с `"-"` в левых колонках пар, числами в правых, и `"-"` в дельте. В подзаголовке row-5 prev отображается `"-"` тогда и только тогда, когда **ни у одной** строки отчёта нет prev (иначе показывается диапазон только по тем, где `PrevAt` есть).

## Расширение списка резервуаров

Если в `reservoir_flood_config` сейчас N включённых организаций, отчёт выведет N блоков. Чтобы добавить водохранилище:

```bash
curl -X POST http://localhost:9010/reservoir-flood/config \
  -H 'Authorization: Bearer $JWT' \
  -H 'Content-Type: application/json' \
  -d '{"organization_id": 96, "sort_order": 1, "is_active": true}'
```

`sort_order` определяет порядок в отчёте.

## Подпись оператора

Внизу отчёта генератор пишет `ShortenName(claims.Name)` (`Иванов Иван Иванович` → `И. Иванов`) в ячейку `N{signer_row}` (top-left of the `N9:R9` merge in the template). `signer_row` сдвигается вниз вместе с количеством блоков: для 9 резервуаров — ряд 25.

## Примеры

```bash
# Сводка за 00:00 (стандартный режим)
curl -H "Authorization: Bearer $JWT" \
  "http://localhost:9010/reservoir-flood/export?date=2026-05-04&format=excel" \
  -o sel.xlsx

# Произвольный час, PDF
curl -H "Authorization: Bearer $JWT" \
  "http://localhost:9010/reservoir-flood/export?date=2026-05-04&hour=15&format=pdf" \
  -o sel-15.pdf
```

## Зависимости

- PDF-конвертация требует наличия `soffice` (LibreOffice headless) на хосте — тот же паттерн, что в остальных Excel/PDF endpoints.
- Шаблон загружается из `template/sel.xlsx`, путь конфигурируется через `Config.TemplatePath`.
