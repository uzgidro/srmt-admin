# Excel templates moved into the binary — deployment notes

## Что изменилось

8 xlsx-шаблонов отчётов теперь **встроены в бинарь** через `go:embed`
(новый пакет `internal/lib/service/excel/templates`). Корневая папка
`template/` удалена из репозитория.

## Что это значит для прода

- **Тома `./template:/app/template:ro` в `docker-compose.yml` больше не нужны.**
  Код туда не смотрит. При следующем обновлении compose можно удалить эту
  строку. Если оставить — безвредно (том просто висит мёртвым).
- **Старые версии шаблонов на хосте больше не влияют на отчёты.** Источник
  правды — код в репозитории. Чтобы изменить шаблон, его правят и пушат как
  обычный код, после чего пересобирают образ.
- **Прод-конфиг (`prod.yaml`):** ключ `template_path` удалён из структуры
  Config — cleanenv молча проигнорирует оставшийся ключ, ошибки не будет.
  Чистить вручную не обязательно, но желательно — потому что новый ключ
  `template_override_path` для прода **не задаётся** (пустая строка = только
  embed, это нужное поведение).

## Что это значит для разработки

- Файлы шаблонов теперь в `internal/lib/service/excel/templates/*.xlsx` —
  правьте их там. После правки нужен `go run ./cmd` или `make dev`.
- **Hot-reload без пересборки** возможен: в `local.yaml`/`dev.yaml` укажите
  ```yaml
  template_override_path: "./some/dir"
  ```
  и кладите туда отредактированные .xlsx **под тем же именем**, что встроенный
  файл (например `sel.xlsx`). Генераторы предпочтут файл из этой папки.
  Если файла там нет — silent fallback на встроенную копию.
- При старте, если override-путь указан, но в папке нет ни одного шаблона —
  в лог пишется `Warn` (страховка от опечатки в пути).

## Override на проде (не рекомендуется)

Можно: установить `template_override_path: "/some/dir"` в `prod.yaml`,
положить туда .xlsx, перезапустить контейнер. Но теряется единый источник
правды. Если делать — **выберите путь, ОТЛИЧНЫЙ от `/app/template`**, чтобы
исключить случайный конфликт со старым ro-томом.

## Размер бинаря

Вырос примерно на сумму размеров 8 .xlsx (~700KB-1MB). Не существенно.

## Откат

Не предусмотрен в виде down-миграции (это код, не БД). Если нужно срочно
вернуться — деплоить предыдущий релиз бинаря.
